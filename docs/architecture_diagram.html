üîç Convolution Layer
Slides a 5√ó5 "window" across the image to detect patterns like edges, curves, and lines. 32 different filters learn to detect 32 different features.
Why Convolution?
Unlike scanning every pixel individually, convolutions detect local patterns regardless of where they appear in the image.

Normalize & Activate
‚ö° Normalization + Activation
BatchNorm: Standardizes the values to prevent them from getting too large or small during training.

ReLU: "Rectified Linear Unit" ‚Äî if the value is negative, make it zero. Otherwise, keep it. This adds non-linearity.
Why needed?
BatchNorm speeds up training 2-3x. ReLU lets the network learn complex, non-linear patterns instead of just straight lines.

üìâ Max Pooling
Takes each 2√ó2 region and keeps only the maximum value. This shrinks the image by half (28‚Üí14) while keeping the strongest features.
Why pool?
Reduces computation, prevents overfitting, and makes the network less sensitive to small shifts in the image.

The second conv layer detects higher-level patterns by combining the edges and curves from layer 1. Now with 64 filters to capture more complex shapes like loops, corners, and digit parts.

Deeper layers need to represent more abstract concepts. A "7" is made of many edge combinations ‚Äî 64 filters can capture this variety.

üé≤ Dropout (Regularization)
During training, randomly "turns off" 25% of the feature maps. This forces the network to not rely too heavily on any single feature.
Why drop neurons?
Prevents overfitting ‚Äî the model can't memorize training data if parts of it are randomly disabled. Like training with a handicap!

üìê Flatten Operation
Reshapes the 3D feature maps (64 √ó 7 √ó 7) into a single 1D vector of 3,136 numbers. This bridges the spatial conv layers to the dense classification layers.
Why flatten?
Fully connected layers expect a 1D input. Flattening preserves all the information but reorganizes it into a format dense layers can process.
